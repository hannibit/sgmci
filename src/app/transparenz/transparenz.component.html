Transparenz

Transparenz in Human Automation Interaction:

Die Fähigkeit eines Systems, dem Benutzer verständlich zu machen, wie es Entscheidungen trifft und welche Informationen es dafür verwendet. Das beinhaltet Einblicke in:

Intentionen: Warum führt das System eine bestimmte Aktion aus?

Leistung: Wie gut funktioniert das System und wie zuverlässig sind seine Ergebnisse?

Zukünftige Pläne: Was plant das System als nächstes zu tun?

Denkprozess: Wie kommt das System zu seinen Entscheidungen?

Für wen ist Transparenz relevant?

Transparenz ist für alle Benutzer von Automationssystemen wichtig, insbesondere aber für:

Unerfahrene Benutzer: Die sich mit der Funktionsweise des Systems nicht auskennen und leicht überfordert sein können.

Benutzer in sicherheitskritischen Umgebungen: Die sich auf die korrekte Funktion des Systems verlassen müssen und im Notfall schnell und richtig reagieren müssen.

Do's

Verwenden Sie eine einfache, klare und für den Benutzer verständliche Sprache. Vermeiden Sie Fachbegriffe und komplexe Formulierungen.

Verwenden Sie Diagramme, Grafiken und Animationen, um komplexe Informationen zu veranschaulichen.

Liefern Sie nur Informationen, die für den Benutzer in der aktuellen Situation relevant sind.

Geben Sie dem Benutzer kontinuierlich Feedback über den Status des Systems und seine Entscheidungen.

Ermöglichen Sie dem Benutzer, das System zu hinterfragen, Feedback zu geben und Entscheidungen zu beeinflussen.

Don'ts

Überfordern Sie den Benutzer nicht mit zu vielen Informationen.

Verbergen Sie die Funktionsweise des Systems nicht vor dem Benutzer.

Lassen Sie den Benutzer nicht im Unklaren über den Status des Systems und seine Entscheidungen.

Gestalten Sie die Kommunikation zwischen Benutzer und System nicht einseitig.

Do: Ein autonomes Fahrzeug zeigt dem Benutzer auf einem Display an, welche Objekte es in seiner Umgebung wahrnimmt und wie es diese klassifiziert

Dont: Ein autonomes Fahrzeug führt eine Notbremsung durch, ohne dem Benutzer den Grund dafür mitzuteilen.

Entwicklungsprinzipien

Das 5-Ebenen-Modell der Transparenz (Mercado et al., 2016)

Definieren Sie, welche Informationen auf welcher Ebene der Transparenz für den Benutzer relevant sind:

Ebene 0: Keine Transparenz: Der Benutzer erhält keine Informationen über das System.

Ebene 1: System Status: Der Benutzer erhält grundlegende Informationen über den Status des Systems.

Ebene 2: Absichten und Ziele: Der Benutzer erfährt, welche Ziele das System verfolgt.

Ebene 3: Entscheidungen und Aktionen: Der Benutzer sieht, welche Entscheidungen das System trifft und welche Aktionen es ausführt.

Ebene 4: Begründung und Erklärungen: Der Benutzer erhält detaillierte Erklärungen für die Entscheidungen und Aktionen des Systems.

Ein Roboterarm in einer Fabrik zeigt dem Benutzer auf Ebene 3 an, welche Bewegungen er als nächstes ausführen wird.

Adaptive Transparenz (Helldin, 2014)

Passen Sie die Transparenzstufe dynamisch an die Bedürfnisse des Benutzers und die Anforderungen der Situation an.

Aufgaben:

Definieren Sie verschiedene Transparenzstufen für Ihr System.

Entwickeln Sie einen Algorithmus, der die aktuelle Situation (z. B. Komplexität der Aufgabe, Erfahrung des Benutzers, Zeitdruck) analysiert und die passende Transparenzstufe auswählt.

Testen Sie den Algorithmus mit Benutzern, um sicherzustellen, dass er die Transparenzstufe angemessen anpasst.

Ein Navigationssystem im Auto blendet detaillierte Karteninformationen nur dann ein, wenn der Benutzer sich in einer komplexen Verkehrssituation befindet.

Potentielle Probleme

Zu viel Information kann den Benutzer überfordern und das Vertrauen in das System verringern.

Die Bereitstellung detaillierter Informationen kann zeitaufwendig sein und die Effizienz des Systems beeinträchtigen.

Die Implementierung von Transparenz in komplexe Softwaresysteme kann aufwendig sein.

Falls die Entscheidungsfindung im System für den User nicht intuitiv erscheint, kann dies zu Vertrauensverlust führen, obwohl alle Informationen transparent sind.

Benutzer könnten die bereitgestellten Informationen falsch interpretieren, was zu fehlerhaftem Handeln führen kann.
