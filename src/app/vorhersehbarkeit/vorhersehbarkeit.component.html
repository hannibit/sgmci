Vorhersehbarkeit

Vorhersehbarkeit in Human Automation Interaction:

Die Fähigkeit des Systems, dem Benutzer ein klares Verständnis davon zu vermitteln, wie es sich in verschiedenen Situationen verhalten wird. Vorhersehbare Automationssysteme sind leichter zu verstehen, zu kontrollieren und zu vertrauen.

Konsistenz: Das System verhält sich in ähnlichen Situationen immer auf die gleiche Weise.

Transparenz: Der Benutzer versteht die Gründe für die Aktionen und Entscheidungen des Systems.

Erklärbarkeit: Das System kann seine Entscheidungen und Aktionen dem Benutzer verständlich erklären.

Für wen ist Vorhersehbarkeit relevant?

Vorhersehbarkeit ist für alle Benutzer von Automationssystemen relevant, insbesondere aber für:

Unerfahrene Benutzer: Die noch lernen, mit dem System umzugehen und sich auf ein konsistentes Verhalten verlassen müssen.

Benutzer in komplexen Umgebungen: Wo unerwartete Ereignisse eintreten können und das System in der Lage sein muss, angemessen zu reagieren.

Do's

Entwickeln Sie klare Regeln und Muster für das Systemverhalten. Vermeiden Sie unnötige Variationen und Ausnahmen.

Verwenden Sie ein konsistentes Design für die Benutzeroberfläche und die Systemreaktionen. Der Benutzer sollte sich leicht orientieren können und wissen, was ihn erwartet.

Geben Sie dem Benutzer frühzeitig Hinweise auf bevorstehende Aktionen oder Änderungen im Systemverhalten.

Verwenden Sie eindeutige Signale, um den Benutzer über den Systemzustand und seine Aktionen zu informieren.

Don'ts

Vermeiden Sie inkonsistentes Systemverhalten. Der Benutzer sollte sich darauf verlassen können, dass das System in ähnlichen Situationen immer gleich reagiert.

Führen Sie keine Aktionen aus, die der Benutzer nicht nachvollziehen kann.

Vermeiden Sie plötzliche und unerwartete Änderungen im Systemverhalten.

Verwenden Sie keine uneindeutigen oder irreführenden Signale.

Do: Ein autonomes Fahrzeug signalisiert dem Fahrer rechtzeitig, wenn es die Spur wechseln möchte.

Dont: Ein Roboterarm führt plötzlich eine unerwartete Bewegung aus, die den Benutzer erschreckt.

Entwicklungsprinzipien

Mentale Modelle (Lichtenthäler et al., 2012)

Gestalten Sie das System so, dass es den mentalen Modellen des Benutzers entspricht. Der Benutzer sollte die Funktionsweise des Systems intuitiv verstehen können.

Analogien: Verwenden Sie Analogien zu bekannten Softwaresystemen oder Prozessen, um dem Benutzer das Verständnis zu erleichtern.

Konzepte: Stellen Sie sicher, dass das Systemverhalten mit den Erwartungen des Benutzers an die zugrundeliegenden Konzepte übereinstimmt.

Ein Roboter, der Getränke serviert, bewegt sich auf eine Weise, die der eines menschlichen Kellners ähnelt.

Machine Learning (ML)

Trainieren Sie das System mit Machine Learning, um aus Daten zu lernen und sein Verhalten an die Bedürfnisse des Benutzers anzupassen.

Mustererkennung: ML-Algorithmen können Muster im Benutzerverhalten erkennen und das Systemverhalten entsprechend anpassen.

Personalisierung: ML kann verwendet werden, um das System an die individuellen Vorlieben des Benutzers anzupassen.

Ein Musik-Streaming-Dienst lernt aus den Hörgewohnheiten des Benutzers und schlägt neue Musik vor, die ihm gefallen könnte.

Potentielle Probleme

Die Implementierung von vorhersehbarem Verhalten kann in komplexen Softwaresystemen schwierig sein.

Zu starre Regeln und Muster können die Flexibilität des Systems einschränken.

Es ist unmöglich, alle potenziellen Situationen im Voraus zu berücksichtigen. Das System muss in der Lage sein, auch in unerwarteten Situationen angemessen zu reagieren.

ML-Algorithmen können durch die Trainingsdaten beeinflusst werden und unerwünschte Verzerrungen (Bias) im Systemverhalten erzeugen.
